# Plan: Remove _comment from benchmark.json and enforce schema exclusivity

## Goal

Remove the `_comment` field from `BenchmarkRunner.java` benchmark.json output (a schema violation)
and update the `persistArtifactsWritesBenchmarkJson` test to assert field exclusivity, preventing
undocumented fields from slipping through undetected in the future.

## Type

bugfix

## Parent Requirements

None

## Background

M582: `BenchmarkRunner.java` line 530 writes a `_comment` field to benchmark.json output. The
persist-artifacts schema in `plugin/concepts/skill-benchmarking.md` defines exactly five fields:
`session_id`, `phase`, `timestamp`, `skill`, and `test_cases`. The `_comment` field is not among them.
JSON has no comment standard; `_comment` is an unofficial workaround with no schema backing.

The test gap (no exclusivity assertion in `persistArtifactsWritesBenchmarkJson`) allowed the violation
to persist undetected. Two committed benchmark.json files in `plugin/skills/` contain the field.

## Approach

Minimal surgical fix:
1. Remove the `_comment` field write from `BenchmarkRunner.java` line 530
2. Update `persistArtifactsWritesBenchmarkJson` to assert the JSON object contains ONLY the five
   documented fields (no extras)
3. Remove `_comment` from existing benchmark.json files in `plugin/skills/`
4. Add a note to `plugin/concepts/skill-benchmarking.md` that benchmark.json is auto-generated by
   BenchmarkRunner

## Risk Assessment

- **Risk Level:** LOW
- **Concerns:** None — additive test assertion, removal of undocumented field
- **Mitigation:** All tests pass after changes

## Execution Plan

### Wave 1: Remove _comment from BenchmarkRunner.java and fix existing files

**Step 1:** In `client/src/main/java/.../BenchmarkRunner.java` line 530, remove the statement that

**Step 4:** In the `persistArtifactsWritesBenchmarkJson` test method, after asserting expected fields

**Step 5:** Run `mvn -f client/pom.xml test` and verify all tests pass.
