# Plan

## Goal

Update `cat:instruction-builder-agent` to produce a `benchmark/test-cases.json` (LLM-orchestrated)
instead of Bats-based e2e tests when building or validating skill instructions. The generated test
cases must follow the organic benchmark design standard (`plugin/concepts/benchmark-design.md`).

## Pre-conditions

- `plugin/concepts/benchmark-design.md` exists (created by `2.1-redesign-benchmark-organic-usage`)

## Post-conditions

- [ ] `plugin/skills/instruction-builder-agent/first-use.md` updated so the step that previously
  created a Bats test now creates `benchmark/test-cases.json` in the skill's directory instead
- [ ] The test-case generation step instructs the agent to:
  - [ ] Follow `plugin/concepts/benchmark-design.md` (organic standard, no priming)
  - [ ] Include at least 2 positive cases (Tier 1: skill chosen; Tier 2: skill followed correctly)
  - [ ] Include at least 3 negative cases (clearly out-of-scope prompts)
  - [ ] Run `empirical-test-runner` to validate ≥80% pass rate before committing
- [ ] No Bats test file is generated by instruction-builder
- [ ] The updated first-use.md passes instruction-builder benchmark validation

## Files Modified

- `plugin/skills/instruction-builder-agent/first-use.md` — test generation step updated

## Execution Steps

### Step 1 — Read current instruction-builder instructions

Read `plugin/skills/instruction-builder-agent/first-use.md` to find the step that generates e2e
tests and understand the current Bats-based approach.

### Step 2 — Identify the test generation step

Locate the step number and exact content describing Bats test creation. Note the step number so
subsequent steps can be correctly renumbered if needed.

### Step 3 — Replace Bats step with benchmark/test-cases.json step

Replace the Bats test creation step with instructions to:
1. Read `plugin/concepts/benchmark-design.md` for the organic test design standard
2. Design test cases for the skill being built:
   - Positive cases: realistic work prompts that require the skill; verify skill chosen (Tier 1)
     and skill followed correctly (Tier 2)
   - Negative cases: clearly out-of-scope prompts (wrong tool count, search-only, explicit paths)
3. Write `plugin/skills/<skill-name>/benchmark/test-cases.json` with no system_reminders
4. Run `empirical-test-runner` to validate ≥80% on positive cases; revise if needed
5. Commit with `test:` prefix

### Step 4 — Renumber steps if needed

If any subsequent steps reference step numbers, update them.

### Step 5 — Commit

```bash
git add plugin/skills/instruction-builder-agent/first-use.md
git commit -m "refactor: update instruction-builder to generate organic benchmark test-cases.json"
```

## Success Criteria

- `plugin/skills/instruction-builder-agent/first-use.md` no longer contains Bats test generation
- The replacement step references `plugin/concepts/benchmark-design.md`
- The replacement step requires empirical validation before committing
- No Bats file path appears in the updated instructions
